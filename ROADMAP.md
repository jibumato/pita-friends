## Phase 0 — 基盤構築 ✅

- ✅ コアフロープロト実装（全14画面 + 誘う/通報シートを状態機械で接続）
- ✅ デザイントークン定数化（色/角丸/ハードシャドウ/タイポ/端末寸法）
- ✅ 共通コンポーネント4種（Avatar / PitaButton / Badge / Chip）
- ✅ 演出（scrIn / popIn / ringSpin / floaty / conf）

## Phase 1 — フロントエンド完成 🚧

プロトタイプを「触れるアプリ」に近づける。バックエンド未接続でもモック駆動で全画面を回せる状態にする。

- ✅ 周辺画面：さがす / 募集掲示板 / 募集作成 / マイページ / 設定 / 安全センター / 通知一覧
- 🚧 状態網羅：空 / ローディング / エラー は実装済み。深夜オフライン / 送信失敗 は未着手
- 🚧 ライト / ダーク テーマ切替（設定にトグルUIあり。全画面テーマ適用は未実装）
- ✅ ルーティング & 下部タブの実接続（画面間ナビゲーション）
- ✅ モックデータ層（`src/data/mock.ts` に分離）
- 🚧 アクセシビリティ（reduced-motion 対応済み。フォーカス可視化は今後）

## Phase 2 — モバイル基盤 ⬜

- ⬜ 配信形態の決定：PWA化 か React Native / Expo 移行
- ⬜ プッシュ通知（合流リマインド等）
- ⬜ カメラ連携（本人確認の書類・顔写真撮影）
- ⬜ 端末ストレージ / セッション永続化

## Phase 3 — バックエンド & データ ⬜

- ⬜ API 設計 & DB 設計（ユーザー / マッチ / 約束 / レビュー / 通報）
- ⬜ 認証（サインアップ・ログイン・セッション・トークン）
- ⬜ リアルタイム基盤（チャット・オンライン状態）— WebSocket
- ⬜ マッチングエンジン（相性スコア算出ロジック）
- ⬜ 通知配信基盤

## Phase 4 — 安全・信頼（コア差別化） ⬜

ピタフレの中核価値。ここの完成度がサービスの信頼を決める。

- ⬜ 本人確認（eKYC）ベンダー選定・統合
- ⬜ 年齢確認（18歳未満の排除）
- ⬜ マナースコア / 信用スコア / ドタキャン率の算出・反映
- ⬜ 通報 → 運営審査 → 利用制限のオペレーションフロー
- ⬜ 外部アプリ誘導・金銭要求の検知（NGワード / パターン検知）
- ⬜ ブロック機能

## Phase 5 — 法務・コンプライアンス ⬜

- ⬜ 利用規約 / プライバシーポリシー / 特定商取引法表記
- ⬜ 個人情報保護・本人確認書類の暗号化保存
- ⬜ 年齢確認の義務対応
- ⬜ （課金導入時）決済・返金・特商法対応

## Phase 6 — 品質保証 ⬜

- ⬜ テスト整備（ユニット / E2E / 負荷）
- ⬜ セキュリティ診断・脆弱性チェック
- ⬜ クローズドβテスト（限定ユーザー）

## Phase 7 — リリース準備 ⬜

- ⬜ CI/CD・監視・ログ・分析基盤
- ⬜ ストア申請（App Store / Google Play）または Web ローンチ
- ⬜ 運営・カスタマーサポート体制・運営マニュアル

## Phase 8 — ローンチ & グロース ⬜

- ⬜ 段階的公開（地域 / 招待制など）
- ⬜ KPI 計測（マッチ成立率 / ドタキャン率 / 継続率）
- ⬜ フィードバックループ・改善サイクル
