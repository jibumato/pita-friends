# ピタフレ — ゲーム仲間マッチングコミュニティ

「ピタフレ」は、今すぐ一緒にゲームを遊べる仲間を安全にマッチングするモバイル体験のアプリです。
中心となるのは **探す → 誘う → 待つ → マッチ成立 → 約束 → 当日合流 → プレイ後レビュー → 信用スコア上昇** という信頼ループ。外部アプリ誘導・アプリ外での直接の金銭要求を排し、アプリ内で完結する安全設計と、マナースコアによる信頼の可視化が特徴です。一緒に遊ぶ時間を提供する「ホスト」への謝礼は、運営が仲介する公式コイン決済のみで安全に行えます。

このリポジトリは、デザインハンドオフ（`.dc.html` プロトタイプ）の **コアフロー** を、React + TypeScript + Vite でピクセル忠実に再現した実装です。

## 技術スタック

- **React 18** + **TypeScript**（関数コンポーネント + Hooks）
- **Vite** 5（開発サーバー / ビルド）
- **Supabase**（Postgres + 認証 + リアルタイム + ストレージ）— バックエンド。未接続でもモックデータのデモモードで動作（セットアップ手順は [`docs/backend-setup.md`](docs/backend-setup.md)）
- スタイルはハンドオフに合わせて **インラインスタイル + デザイントークン** で構成（外部 UI ライブラリ非依存）
- フォント: Google Fonts `DotGothic16`
- アイコン: すべてインライン SVG（依存なし）

> ハンドオフでは React Native / Expo なども候補に挙げられていますが、レビュー・プレビューのしやすさを優先し、端末枠を含むモバイルファーストの Web アプリとして実装しています。デザイントークンと共通コンポーネントは移植しやすい形で分離しています。

## セットアップ

```bash
npm install
npm run dev      # http://localhost:5173 で起動
```

その他のスクリプト:

```bash
npm run build    # 型チェック(tsc) + 本番ビルド
npm run preview  # ビルド成果物のプレビュー
npm run lint     # 型チェックのみ(tsc --noEmit)
```

## コアフロー（状態機械）

ヘッダーに 5 フェーズのフローレール（準備→探す→約束→合流→評価）を表示し、現在地をハイライトします。端末枠は 375×812。全 14 画面 + 2 オーバーレイを 1 本の状態機械で遷移します。

| # | 画面 | キー | 内容 |
|---|---|---|---|
| 1 | ようこそ | `welcome` | オンボーディング入口（PRESS START） |
| 2 | 本人確認 | `verify` | 確認済みバッジ取得動線（STEP 1/2） |
| 3 | プロフィール作成 | `setup` | ニックネーム・ゲーム・プレイスタイル（STEP 2/2） |
| 4 | ホーム | `home` | いま遊べる横スクロール + 今夜のおすすめマッチ + 下部タブ |
| 5 | プロフィール | `profile` | スタットタイル・あそぶゲーム・時間バー・レビュー |
| 6 | 誘う（ボトムシート） | `invite` | ゲーム/日時チップ・メッセージ・安全注意帯 |
| 7 | 返事待ち | `sending` | リング回転スピナー + 浮遊アバター（`autoAdvanceMs` 後に自動遷移） |
| 8 | MATCH FOUND! | `match` | ラベンダー背景の感情ピーク（`popIn` 演出・相性スコア） |
| 9 | PARTY成立 | `party` | 紙吹雪 + 3人パーティ + カウントダウン |
| 10 | トークルーム | `talk` | チャット・約束カード（確定でフレンドコード解放）・通報起動 |
| 11 | 合流リマインド | `reminder` | 開始直前通知 |
| 12 | 合流中 | `joining` | ロビー合流ローディング（自動遷移） |
| 13 | プレイ後レビュー | `review` | ★評価 + 良かった点タグ |
| 14 | RESULT | `result` | 紙吹雪 + 信用スコア上昇演出（★4.90→4.92）・称号解放 |
| — | 通報・ブロック | overlay | 外部誘導/金銭/暴言/なりすましの通報理由 |

### デモ調整パラメータ（`src/App.tsx`）

- `MATCH_SCORE`（相性%, 既定 92）
- `AUTO_ADVANCE_MS`（自動遷移速度, 既定 2400ms）
- `SHOW_RAIL`（フローレール表示, 既定 true）

## ディレクトリ構成

```
src/
  theme/tokens.ts        デザイントークン（色/角丸/影/タイポ/端末寸法）
  flow.ts                状態機械定義（画面キー・ステップ対応・紙吹雪生成）
  hooks/usePress.ts      押し込みホバー表現
  components/            再利用コンポーネント
    Avatar / PitaButton / Badge / Chip   ← ハンドオフの共通4種
    Icon / StatusBar / FlowRail / PhoneFrame / BottomTabs / Confetti / Screen
  screens/               14画面 + Invite/Report ボトムシート
  App.tsx                状態機械 + 外側レイアウト（ヘッダー/レール/端末枠）
```

## デザイントークン（抜粋）

ネオブルータリズム風。全要素に `1.5px solid` インク色ボーダー + ぼかし無しのハードシャドウ。ペール（パステル）基調。

| 役割 | Hex |
|---|---|
| インク（ボーダー/本文/影） | `#453D5C` |
| ラベンダー（プライマリ） | `#A78BDF` |
| ペールライム（肯定/報酬） | `#E4F0A0` |
| ライト背景 | `#F7F6FB` |

全値は `src/theme/tokens.ts` を参照。
