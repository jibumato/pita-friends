# 弁護士レビュー依頼パッケージ

このファイルは、弁護士への相談時に**そのまま渡す**ことを想定した論点整理です。
(作成: 2026-07-21。サービス実装と法務ドラフトの現状を反映)

## 1. サービス概要

- **名称**: ピタフレ(ゲーム仲間マッチングサービス)
- **内容**: オンラインゲームを一緒に遊ぶ相手を探すWebアプリ(PWA)。
  本人確認済みユーザーのみ利用可(18歳未満は排除)。女性の安全を重視した設計
  (誘いを受ける範囲の本人コントロール・承認制・通報/ブロック・メッセージの自動みまもり)。
- **収益化**: サービス内通貨「コイン」(1コイン=1円、有効期限180日)をStripeで販売。
  ユーザー(ゲスト)はコインを消費して、ホスト(時給制で「一緒に遊ぶ時間」を提供する
  ユーザー)を予約する。
- **明示的に出会い系ではない**: 恋愛・出会い目的の利用は規約で禁止し、検知・通報体制あり。
- **事業者**: 【個人事業主/法人・確定後に記載】

## 2. 資金フロー(現在の実装)

```
[ゲスト] ─¥(Stripe)─▶ [当社] コイン付与(purchase balance)
[ゲスト] ─コイン消費─▶ 予約成立(この時点でホストには何も渡らない=エスクロー)
[ゲスト] ─「プレイ完了」確定(または72時間で自動確定)─▶
          [ホスト] 報酬コイン(earned_balance)が確定
[ホスト] ─換金申請(1,000コイン以上、手数料300コイン控除)─▶
[当社] ─月末締め翌月払いで、当社口座からホスト名義口座へ銀行振込(総合振込)─▶ [ホスト]
```

- 購入コイン(balance)と報酬コイン(earned_balance)は**別会計**。
  **購入コインは換金不可**(現金化ルートは役務提供の対価のみ)。
- 直前キャンセル(開始1時間前以降のゲスト都合)は返金せず、当該コインは
  ホストの報酬になる(機会損失の補償)。
- ボーナスコイン: コインパックに無償の上乗せあり(例: ¥1,000で1,050コイン)。

## 3. 最優先で確認したいこと(換金まわり)

**Q1. 当社が自社口座からホストへ銀行振込する方式は、資金移動業の登録なしで
適法に運営できるか。**
- 「ホストの役務対価をゲストから預かり、検収後にホストへ引き渡す」構造を
  **収納代行**として整理できるか。
- 参考: 同種サービス「GameRoom」(株式会社ゲームトレード)は、コイン購入
  +出品者への清算金支払い(銀行振込)を運営しており、資金移動業登録の
  有無は不明。
- 決済代行(Stripe Connect)を使う代替案は実装済みでいつでも切替可能
  (コスト面で自社振込を選好。詳細: coin-economy-legal-review.md §7.3)。

**Q2. 収納代行と整理する場合、現在の設計で不足している要素はあるか。**
- コインが「特定の取引に紐づかない汎用前払い残高」である点の影響
- エスクロー(検収確定後にのみホスト残高化)である点は有利に働くか
- 資金の分別管理(ユーザー資金と事業資金の口座分離)はどこまで必要か

**Q3. ボーナスコイン(無償付与分)が報酬経由で換金されうる点の扱い。**
- 当社の受領額を超える払い出しが理論上ありうる。手数料(300コイン/回)で
  実質相殺される見込みだが、制度設計として問題ないか。

**Q4. 換金手数料(300コイン=300円/回)の法的性質。**
- 振込実費への充当として整理してよいか。表示上の注意はあるか。

**Q5. 犯罪収益移転防止法・税務。**
- 当社に取引時確認義務は生じるか(資金移動業非該当なら不要という理解でよいか)
- ホストへの支払いについて、支払調書等の税務上の義務はあるか

## 4. 次に確認したいこと

**Q6. 出会い系サイト規制法(インターネット異性紹介事業)への非該当性。**
- ゲーム目的限定・異性交際目的の禁止・通報体制ありだが、
  「金銭を介して特定の相手と1対1の時間を過ごす」予約機能があるため、
  実態判断で該当しないか確認したい(詳細: coin-economy-legal-review.md §4.2)。
- 風営法(接待飲食等営業・無店舗型性風俗特殊営業等)への該当リスクはないか。

**Q7. 前払式支払手段(資金決済法)。**
- 自家型としての整理・表示(ドラフトあり)・基準日残高1,000万円超で届出+供託、
  という理解の確認。180日有効期限の設定は妥当か。

**Q8. 労働法・下請法等。**
- ホストは個人の役務提供者(雇用ではない)という整理でよいか。
  規約上の手当てが必要か。

**Q10. コイン有効期限の設計はどちらを推奨するか。**〔※初回回答(Q7)を受けた追加質問〕
- 案A: 「発行日から起算して6月を経過する日の前日まで」として資金決済法の
  **適用除外**を取る(表示・届出・供託が不要。期限延長は不可)
- 案B: 180日(または期限自由)の通常の自家型として整理(基準日残高1,000万円超で届出+1/2供託)
- 小規模スタートの当社にとってどちらが実務上安全か、推奨と、案Aの場合の
  規約文言・適用除外が外れる落とし穴(期限延長キャンペーン等)を確認したい。

**Q9. ドラフト4点のレビュー**(いずれも施行前ドラフト・【】は未確定):

  第9条(キャンセル・72時間自動確定)・第14条(免責)
