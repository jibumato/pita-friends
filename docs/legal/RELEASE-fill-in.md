# 公開前に記入する項目（法務ページ）

技術・方針で確定できる `【　】` はすべて埋めました。残っているのは**あなたの実情報が必要な項目だけ**です。
公開直前に、下の表のとおり各ファイルの該当箇所を置換してください（多くは同じ文言なので一括置換でOK）。

> 事業形態は「**個人事業主**」で確定（2026-07-23）。事業者名＝あなたの本名になります。

## 一括で置換すると早いプレースホルダ

| プレースホルダ（現在の文言） | 何を入れるか | 出てくる場所 |
|---|---|---|
| `【氏名（個人事業主）：公開前に記入】` | あなたの本名（個人事業主の氏名） | 規約・プライバシー・特商法・資金決済法表示 |
| `【サービス専用の問い合わせ用メールアドレス：公開前に記入】` | 用意した専用の問い合わせ用メール（例：`support@…`） | 規約・プライバシー・特商法・資金決済法表示 |
| `【独自ドメイン取得後に記入（例：https://pitafure.jp）】` | 取得した独自ドメインの本番URL | 特商法（販売URL） |

## 個別に判断が要る項目

| 箇所 | 現状 | 確定のトリガー |
|---|---|---|
| 特商法の **所在地・電話番号** | 〔弁護士確認まで保留〕 | 弁護士Q9で「請求により遅滞なく開示」運用の可否を確認 → 自宅かバーチャルオフィス等かを決めて記入 |
| プライバシーの **所在地** | 〔弁護士確認まで保留〕 | 上と同じ判断に合わせる |
| **制定日** | 〔施行日に記入〕 | 弁護士レビュー完了後、実際に公開する日付を記入 |

## 反映のしかた

- 規約・プライバシー・特商法は `docs/legal/*.md` を編集すれば、**アプリ内の表示ページも自動で更新**されます（`src/content/legalDocs.ts` が同ファイルを読み込んでいるため二重管理不要）。
- 資金決済法表示（発行者・苦情窓口）は `src/content/legalDocs.ts` 内に直接あります。
- 編集後は `npm run build` → コミット → デプロイで本番反映。

## まだやってはいけないこと

- **制定日を「今日」で埋めない**（弁護士レビュー前の施行になってしまう）。
- **住所・電話を先に自宅で確定しない**（弁護士の開示運用の可否を待つ）。
- **換金機能の有効化**は弁護士OKまで行わない（特商法・規約に注記済み）。
