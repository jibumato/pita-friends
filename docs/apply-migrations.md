# マイグレーションの適用手順（あなたの手元で実行）

このリポジトリを開発しているサンドボックス環境は、ネットワークポリシーによりあなたのSupabaseプロジェクトへ接続できません。そのため、**マイグレーションの適用はあなたの手元で行ってください。** 以下のいずれかの方法で数分で完了します。

---

## 方法A: SQL Editorに貼り付け（最も簡単・CLI不要）

1. Supabaseダッシュボード → 左メニューの **SQL Editor** → **New query**
2. `supabase/schema-all.sql`（`0001`〜`0007`を1つに結合したファイル）の中身を全部コピーして貼り付け
3. **Run** をクリック

エラーなく完了すれば、全テーブル・RLS・関数・Storageバケットが作成されます。

> 注: `schema-all.sql` は `supabase/migrations/` の7ファイルを番号順に連結しただけのものです。個別に実行したい場合は、`0001` → `0007` の順に1ファイルずつ実行しても同じ結果になります。

---

## 方法B: Supabase CLI（`supabase db push`）

CLIをすでにお使いなら、こちらの方が今後のマイグレーション管理が楽です。

```bash
npm install -g supabase        # 未インストールなら
supabase login                 # ブラウザでトークン認証
supabase link --project-ref ytxsyklnqepanloomzcw
supabase db push               # supabase/migrations/ を番号順に適用
```

---

## 適用後にやること

### 1. アプリを起動して接続確認

`.env.local` はこのリポジトリに用意済みですが、`.gitignore` 対象なので**あなたの手元には無い場合があります**。無ければ以下を作成してください（値はSupabaseの `Settings > API`）。

```
VITE_SUPABASE_URL=https://ytxsyklnqepanloomzcw.supabase.co
VITE_SUPABASE_ANON_KEY=（anon public キー）
```

```bash
npm install
npm run dev
```

ブラウザのコンソールに `デモモードで起動します` が**出なければ**接続成功です。「PRESS START」からメール＋パスワードでアカウントを作成できます。

> Supabaseの `Authentication > Providers > Email` で「Confirm email」が有効な場合、サインアップ後に確認メールのリンクを踏む必要があります。テストを簡単にしたい場合は一時的にオフにできます。

### 2. 自分を管理者に登録（本人確認の審査に必要・初回のみ）

先にそのアカウントで一度サインアップ／ログインしてから、SQL Editorで:

```sql
insert into public.admins (user_id)
select id from auth.users where email = '<あなたのログインメールアドレス>';
```

以降、そのアカウントの **設定画面 → 管理者メニュー → 本人確認の審査** から、提出された本人確認を承認/却下できます。

### 3. 動作確認の流れ

1. アカウントA（あなた／管理者）でログイン
2. アカウントB（テスト用・別メール）でサインアップ → 本人確認で書類・顔写真を提出（テストなので手元の画像でOK）
3. アカウントAの管理画面で承認
4. アカウントBで「ホストになる」→ 掲載 → アカウントAの「さがす」にアカウントBが表示される → 予約

これで、認証・プロフィール・本人確認・ホスト掲載・さがす・予約（コイン消費）まで実データで一気通貫に動くことを確認できます。

---

## 私（サンドボックス側）でできること・できないこと

- ✅ できる: コードの修正、マイグレーションSQLの追加・修正、型チェック、ビルド、デモモード（バックエンド未接続）での画面確認
- ❌ できない: あなたのSupabaseへの接続を伴う操作（マイグレーション適用、実データでの動作確認）— ネットワークポリシーで遮断されているため

適用中にエラーが出た場合は、そのエラーメッセージを教えていただければ、SQLを修正します。
